# Trips

Trips are planned field deployments.

## Key Fields

- Name
- Start and end dates
- Trip Leader
- Team members
- Status
- Notes
- Submitted and approval metadata

Trip dates must be ordered and cannot span more than one calendar year.

## Statuses

- Draft
- Submitted
- Approved
- Completed
- Cancelled

Completed and cancelled Trips are terminal. Normal editing, Site Visit changes,
Job assignment changes, and Requirement changes are frozen.

## Related Objects

Trips contain Site Visits. Site Visits can have assigned Jobs. Trip closeout
sets final outcomes for Site Visits and assigned Jobs.
